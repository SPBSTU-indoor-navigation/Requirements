# Список требований

## Описание
Требования разделены на версии: текущую разрабатываемую и будущее обновление. Требования на будущую версию можно реализовать в текущей, однако **необязательно**.
Любой функционал не описанный в требованиях, но необходимый для работы приложения, реализуется разработчиком на его усмотрение.

## Обновление статуса
Обновления статуса выполнения требований производится путём PullRequest'a, который принимается после одобрения (approve) одним из участников проекта. Утверждение PullRequest'a ставится после проверки соответствия тестовой версии указанному требованию.
